Формулировка второй задачи

Ежедневник с задачами
Бот будет добавлять, показывать и отмечать задачи как выполненные.

Пример:

-/start -Добро пожаловать! Я могу организовать ваши задачи.
Команды:
/add-добавить задачу
/tasks-показать список задач
/done-выполнено
/dTask-список выполненных задач
/delete-удалить
/help-помощь

-/add Полить цветы
-Задача добавлена

-/add Накормить кота
-Задача добавлена

-/tasks
-Вот список задач:
Полить цветы
Накормить кота

-/done Полить цветы
-Задача “Полить цветы” отмечена выполненной!

-/dTask
-Вот список выполненных задач:
Полить цветы

-/delete Накормить кота
-Задача “Накормить кота” удалена из задач





Формулировка третьей задачи

Предоставление пользователю возможности выгрузить из тг бота(экспортировать) и импортировать список своих задач (текущих и выполненных) через JSON файл.
Задачи при импорте должны дополнить уже имеющиеся списки в боте
При совпадении задач из списка и из файла, бот убирает повторы и оставляет только одну запись.


В список всех команд добавляются  /export и /import
/export-предоставить список задач пользователя в файле
/import-загрузить список задач из файла

Содержимое файла: сначала текущие задачи, далее выполненные

Пример:
Предположим, что у нас есть списки команд:
Вот список ваших задач:
1.Полить цветы
2.Накормить кошку
Вот список выполненных задач:
1.Помыть пол

Пользователь: -/export “tasks_list”
Бот: -Ваш список задач в виде документа (отправка “tasks_list.json”)

Пример:
Например загружаем тот же файл «tasks_list.json”
Пользователь: /import
Бот: -Отправьте JSON файл со списком задач
Пользователь отправляет файл и его задачи из current_tasks добавляются в список текущих задач, а completed_tasks в список выполненных.
При совпадении задач из списка и из файла, бот убирает повторы и оставляет только одну запись.
Бот: -Задачи успешно добавлены, можете проверить списки с помощью команд /tasks и /dTask
При ошибке: -Что-то пошло не так, отправьте файл еще раз или проверьте его содержимое.


Пример содержимого файла:
*******************tasks_list.json****************************
{
“current_tasks”: [
“Полить цветы”,
“Накормить кошку”
],
“completed_tasks”: [
“Помыть пол”
]
}
********************************************************************
current_tasks - список текущих задач пользователя
completed_tasks - список выполненных задач пользователя


Для поддержки второй платформы добавляем дополнительные команды, с помощью которых будем определять подключенного пользователя для синхронизации работы бота.
Делаем это в самом начале работы пользователя с ботом
/registration - зарегистрироваться (для новых пользователей)
/login - войти в существующий аккаунт
/exit - выйти из аккаунта




Формулировка четвертой задачи

1.Добавляем базы данных в задачу.

Бот будет хранить задачи, даже после перезапуска, посредством использования БД.

2.Добавляем кнопки вместо команд. Бот меняет состояния в зависимости от команд.

Исчезнут привычные команды по типу /add, вместо них появятся кнопки с названиями команд. Бот, при вызове определенных команд будет создавать запросы на ввод задач для определенных действий с ними. Он будет ожидать ответа пользователя именно для вызванной команды.

Пользователь пишет /start и бот показывает меню с кнопками:
[Регистрация]
[Войти в аккаунт]
[Выйти из аккаунта]
[Добавить задачу]
[Показать список задач]
[Выполнено]
[Список выполненных задач]
[Удалить]
[Экспорт]
[Импорт]
[Помощь]

Функционал такой же, как и с командами

Пример:
Пользователь нажимает кнопку [Добавить задачу]
Бот: -Напишите задачу, которую нужно добавить в список
Пользователь: -Полить цветы
Бот: -Задача добавлена

Пользователь нажимает кнопку [Добавить задачу]
Бот: -Напишите задачу, которую нужно добавить в список
Пользователь: -Накормить кота
Бот: -Задача добавлена

Пользователь нажимает кнопку [Показать список задач]
Бот:-Вот список задач:
1.Полить цветы
2.Накормить кота


Пользователь нажимает кнопку [Экспорт]
Бот: -Напишите имя файла для экспорта. Например: ‘list’
“tasks_list”
Бот: -Ваш список задач в виде документа (отправка “tasks_list.json”)

Пользователь нажимает кнопку [Выполнено]
Бот: -Напишите название выполненной задачи
Пользователь: -Полить цветы
Бот: -Задача “Полить цветы” отмечена выполненной!

Пользователь нажимает кнопку [Список выполненных задач]
Бот: -Вот список выполненных задач:
1. Полить цветы

Пользователь нажимает кнопку [Удалить]
Бот: -Напишите название задачи для удаления
Пользователь: -Накормить кота
Бот: -Задача “Накормить кота” удалена из задач

Пользователь нажимает кнопку [Помощь]
Бот показывает меню со списком всех функций

Пользователь нажимает кнопку [Импорт]
Бот: -Отправьте JSON файл со списком задач
Пользователь отправляет файл и его задачи из current_tasks добавляются в список текущих задач, а completed_tasks в список выполненных.


Пример содержимого файла:
*******************tasks_list.json****************************
{
“current_tasks”: [
“Полить цветы”,
“Накормить кошку”
],
“completed_tasks”: [
“Помыть пол”
]
}
