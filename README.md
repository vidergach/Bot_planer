Формулировка третьей задачи

Предоставление пользователю возможности выгрузить из тг бота(экспортировать) и импортировать список своих задач (текущих и выполненных) через JSON файл.
Задачи при импорте должны дополнить уже имеющиеся списки в боте
При совпадении задач из списка и из файла, бот убирает повторы и оставляет только одну запись.


В список всех команд добавляются  /export и /import
/export-предоставить список задач пользователя в файле
/import-загрузить список задач из файла

Содержимое файла: сначала текущие задачи, далее выполненные

Пример:
Предположим, что у нас есть списки команд:
Вот список ваших задач:
1.Полить цветы
2.Накормить кошку
Вот список выполненных задач:
1.Помыть пол

Пользователь: -/export “tasks_list”
Бот: -Ваш список задач в виде документа (отправка “tasks_list.json”)

Пример:
Например загружаем тот же файл «tasks_list.json”
Пользователь: /import
Бот: -Отправьте JSON файл со списком задач
Пользователь отправляет файл и его задачи из current_tasks добавляются в список текущих задач, а completed_tasks в список выполненных.
При совпадении задач из списка и из файла, бот убирает повторы и оставляет только одну запись.
Бот: -Задачи успешно добавлены, можете проверить списки с помощью команд /tasks и /dTask
При ошибке: -Что-то пошло не так, отправьте файл еще раз или проверьте его содержимое.


Пример содержимого файла:
*******************tasks_list.json****************************
{
“current_tasks”: [
“Полить цветы”,
“Накормить кошку”
],
“completed_tasks”: [
“Помыть пол”
]
}
********************************************************************
current_tasks - список текущих задач пользователя
completed_tasks - список выполненных задач пользователя


Для поддержки второй платформы добавляем дополнительные команды, с помощью которых будем определять подключенного пользователя для синхронизации работы бота.
Делаем это в самом начале работы пользователя с ботом
/registration - зарегистрироваться (для новых пользователей)
/login - войти в существующий аккаунт
/exit - выйти из аккаунта




Формулировка четвертой задачи

1.Добавляем базы данных в задачу.

Формулировка шестой задачи

Подключение чата GPT для возможности добавления подзадач.
Добавляем команду /add_subtasks_with_gpt и кнопку [GPT добавление подзадач] в меню для расширения задач.

Когда пользователь выбирает задачу для расширения, он может нажать на [GPT добавление подзадач], бот уточнит детали и пожелания пользователя по этой задаче и отправит формулировку в чат GPT. После этого будет выведен список подзадач, сгенерированный GPT. Решение об их сохранении или удалении происходит ДО сохранения в списке пользователя.

После того, как поступает ответ от чата GPT с подзадачами, пользователь принимает решение сохранить их или удалить. Для этого нужно использовать /save_subtasks_from_gpt кнопка [Сохранить] или /delete_subtasks_from_gpt кнопка [удалить]

Пример:

Пользователь нажимает  [расширить задачу]
Бот: - Выберите задачу, которую хотите расширить.
Полить цветы
Нарисовать картину
Пользователь: - 2
Бот: -Отлично! Выберите действие, которое хотите сделать:
/add_subtasks_with_gpt - добавить подзадачи с помощью чата GPT
/add_subtask - добавить подзадачу
/delete_subtask - удалить подзадачу
/edit_subtask - изменить подзадачу
/finish_expand - окончить расширение задачи

Пользователь нажимает  [GPT добавление подзадач]
Бот: -Напишите детали и пожелания по задаче, для более точного добавления новых подзадач
Пользователь: -Рисунок красками и кисточками, хочу рисовать природу
Бот: -Подзадачи, написанные чатом GPT:
Налить воду
Найти краски
Выбрать пейзаж
Посмотрите список задач. Если все верно, нажмите [сохранить], в противном случае [удалить]
Если [сохранить]:
Бот: -Подзадачи сохранены, можете посмотреть общий список задач

Пользователь нажимает [окончить расширение], [Показать список задач]
Бот: -Вот список задач:
1.Полить цветы
2.Нарисовать картину
2.1 Налить воду
2.2 Найти краски
2.3 Выбрать пейзаж

Если [удалить]:
Бот: -Подзадачи удалены, при повторном процессе сделайте запрос более точным, чтобы получить желаемые подзадачи
